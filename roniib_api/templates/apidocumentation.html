Authentication

Once you have your API key, include it as an "Authorization" header in your HTTP requests using the Access Token scheme as illustrated in the snippets below.

Rate Limiting

Our API enforces rate limits to ensure fair usage and to protect the system from abuse. Rate limiting is based on the number of requests a client can make within a specific time window.If you exceed your rate limit, you will receive a 429 Too Many Requests response.

Error Handling

The API uses standard HTTP status codes to indicate the success or failure of a request.


Here are some commonly used status codes:

  • 200 OK: The request was successful, and the response contains the requested data.

  • 400 Bad Request: The request was invalid, and the server could not understand it. The response contains details about the error in the response body.

  • 401 Unauthorized: The request requires authentication, and the credentials are missing or invalid.

  • 403 Forbidden: You do not have permission to access the requested resource.

  • 404 Not Found: The requested resource does not exist on the server.

  • 500 Internal Server Error: An unexpected error occurred on the server. You should not retry the request until the issue is resolved.

Error responses typically include a JSON object with information about the error.

FAQ and Support

Frequently Asked Questions (FAQs)
Q: What is the API's base URL?
A: The base URL for our API is https://api.roniib.com.

Q: How do I authenticate with the API?
A: You can authenticate by including your API key in the Authorization header with an Access token. For example: Authorization: Token YOUR_API_KEY.

Q: What response format does the API support?
A: The API supports JSON format by default. You can specify the response format by setting the Accept header to application/json in your request.

Q: Are there any rate limits on API requests?
A: Yes, we have rate limits in place to ensure fair usage.Please refer to our Rate Limiting section at the pricing page for more details.

Q: Where do I get an API Key?
A: API Key is found by navigating to My Account >My API Keys. Copy the API Key and place them in the headers of your requests. In case you would like to change the API Key, click Generate API Key
